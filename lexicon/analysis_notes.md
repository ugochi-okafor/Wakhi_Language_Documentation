# Lexicon Analysis Notes

## 1. Temperature system structure

The Wakhi temperature system is organised as a scalar continuum:

from extreme cold → neutral → warm → hot → extreme heat

Key structure:

- /ik rɑŋʃu/ → extreme cold
- /syr/ → cold
- /ʃlut/ → lukewarm midpoint
- /ʃund/ → warm
- /garm/ → hot
- /mats/ → very hot
- /ɣamats/ → intensified hot
- /ˈtɑːodi mɑts/ → consumption threshold (too hot to drink)

This shows a **graded lexical system with both scalar and threshold-based expressions**.

---

## 2. Change-of-state system

Temperature is not only described statically but dynamically:

- vitk → become cold
- vitɑ / vitja / wost → come into being / become
- peʃt → boiling (liquids)

This indicates a **process-oriented encoding of temperature change**, especially in liquids and cooking contexts.

---

## 3. Domain restriction pattern

A key finding is that temperature lexemes are **domain-bound**:

### Strong usage domains
- environment
- weather
- food and drink

### Weak or absent usage domains
- behaviour
- emotion
- colour

Instead of metaphorical extension, Wakhi uses:
- baghet → unfriendly person
- karha → anger or conflict
- sik → colour system

This shows **lexical replacement rather than metaphorical expansion**.

---

## 4. Degree marking system

The intensifier /ɣaftʃ/ functions independently of temperature lexemes:

- ɣaftʃ mats → very hot
- ɣaftʃ cold contexts (possible extrapolation)

This suggests a **general scalar intensification system not restricted to temperature semantics**.

---

## 5. Key typological insight

Unlike languages such as English, Wakhi does not extend temperature terms into:

- personality descriptions (“cold person”)
- emotional states (“warm heart”)
- abstract evaluation (“heated argument”)

Instead, it maintains **separate lexical domains for social and emotional meaning**.

This makes the system:
- semantically conservative in metaphor
- structurally clear in domain separation
