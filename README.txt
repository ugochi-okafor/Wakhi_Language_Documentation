File structure:
* Full recordings - uncut and partially annotated full recordings (around 30 mins each). It is better to open them in Praat as Long Sound objects.
* Parts - recordings divided into short chapters, fully annotated.

File name conventions:
* date in YYMMDD format
* "S" for Session + session number
* multiple recordings during one session (typically two) are denoted by letters (a, b, etc.)
* "P" for Part + part number
* Name of the part
* Recording metadata is always considered part 0 for consistency

E.g.:
* 260129_S1a.wav - the first recording of the first session, which was held on 29.01.2026
* 260129_S1a_P3_Greetings.vaw - the third part of the first recording of the first session on 29.01.2026, when they talk about Greetings in Wakhi

Text Grid Tiers:
* Part - general part of the conversation
* Speaker - current speaker
* Content - what is being said
* Comments - extra comments

Symbols:
// - broad transcription
[] - narrow transcription
'' - loose translation
"" - quotation, non-Wakhi terms
| - Praat boundary (used only in this file)

E.g.:
| When we meet each other, we say | /ʧɪsxəli/ 'how are you' | which means "how are you" |

Guidelines for annotation:
* All relevant parts are annotated. Irrelevant parts should be cut, left unmarked or marked "--"
* Experimenter's utterances can be annotated loosely
* Sajid's utterances are generally annotated word-by-word
* ALL Wakhi content producend by Sajid is annotated
* No need to annotate other people attempts at speaking Wakhi
* Explanations about meaning can be left unmarked when it is obvious, e.g.:
 - | /wuz/ 'I am' | means "I am" | - redundant
 - | /wuz/ 'I am' |              | - okay
* However, they should be left if they provide extra cultural context

Note: Transcriptions in this file are ad hoc examples and may be wrong.
